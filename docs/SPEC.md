# SPEC · Функциональная спецификация

Документ описывает поведение приложения **1KZARIKI («Зарики»)** — что оно делает с точки зрения пользователя: экраны, навигация, конфигурация правил, ввод и правка счёта, статистика, бэкап и синхронизация. Технические детали (типы, алгоритм подсчёта, хранилище) — в [ARCHITECTURE.md](ARCHITECTURE.md). Правила самой игры — в [RULES.md](RULES.md).

---

## 1. Цель и принципы

Приложение заменяет бумажку при игре в «1000 на кубиках». Оно не бросает кости и не считает комбинации — игроки бросают вживую и **вписывают набранное за кон руками**. Задача приложения — вести таблицу, считать итоги и автоматически отслеживать состояния правил (бочка, ямы, заход, штрафы, обгон, победа).

Принципы:

- **Телефон в первую очередь.** Всё управление — крупными тапами; ввод цифр и букв через экранные модалки.
- **Ничего не теряется.** Состояние пишется в браузер на каждое изменение.
- **Случайно не сломать.** Любое удаление — через диалог подтверждения.
- **Ориентация не важна.** Вёрстка адаптивная, портрет и ландшафт работают одинаково.

## 2. Глоссарий

| Термин | Значение |
|---|---|
| **Кон** | Один раунд: каждый игрок бросает и записывает набранное. Строка таблицы. |
| **Болт** | Кон, закрытый в ноль (нет зачётных костей). |
| **Самосвал (555)** | Счёт встал ровно на 555 — весь счёт сбрасывается в ноль. |
| **Бочка** | Порог (880 или 850), на котором счёт замораживается; нужно закрыться ровно до 1000 за один ход. |
| **Яма** | Промежуточный порог (300, 600), из которого надо выбраться за ход. |
| **Заход** | Открытие счёта: пока не набрал за кон ≥ порога, очки не пишутся. |
| **Перебор** | Превышение 1000 при правиле «ровно 1000» — не засчитывается. |
| **Обгон** | Догнал соперника точно в его счёт — он откатывается назад. |

## 3. Экраны и навигация

Нижняя панель навигации (4 вкладки): **Игры · Правила · Стата · Синк**. Внутри партии вверху — кнопка «‹ К партиям».

### 3.1. Игры (главная)

- Геро-блок и кнопка **«+ Новая партия»**.
- Список партий, отсортированный по последнему изменению. Карточка показывает: имя партии, статус (`идёт` / `на бочке` / `партия сыграна`), участников, текущего лидера и его очки.
- Тап по карточке открывает партию. Иконка **🗑** справа — удаление партии (с подтверждением, в котором указано имя партии).
- Пустое состояние, если партий нет.

### 3.2. Новая партия

- **Игроки.** Список с авто-именами «Игрок N». Тап по имени или ✎ открывает клавиатуру-модалку. Игрока можно удалить (минимум 2). Кнопка «+ Добавить игрока».
- **Правила.** Сначала ряд пресетов (один тап применяет весь набор), затем карточка с отдельными переключателями (см. §4). Изменение любого переключателя сбрасывает выделение пресета.
- Кнопка **«Поехали»** создаёт партию (имя по умолчанию «Партия N») и открывает дашборд.

### 3.3. Дашборд партии

- Заголовок с именем партии и кнопкой «Правила».
- **Баннер хода** — чей сейчас ход (первый игрок с незаполненной клеткой в последнем коне). При победе — баннер «Партия сыграна» с именем победителя.
- **Плашки бочки** — для каждого игрока на бочке: имя, «закрыться за один ход» и большое «−N» (сколько до 1000).
- **Таблица счёта** — колонки игроков, строки конов, шапка с лидером (корона), подвал с итогами `Σ` и статусом каждого игрока.
- **Действия** — «+ Новый кон», «Править коны» (режим удаления конов).

Подробнее ввод и правка — §5.

### 3.4. Правила

Страница правил игры. Если открыта из партии — сверху чипы активных настроек этой партии, а тексты разделов (бочка, вход, обгон, болты, перебор) подставляются под выбранные значения. Без партии показывает дефолтный набор.

### 3.5. Статистика

- Сводка: число партий, выходов на бочку, самосвалов.
- **Зал славы** — игроки (по имени, кросс-партийно), отсортированы по победам: победы, бочки, самосвалы, рекордный кон.
- Пустое состояние, если партий нет.

### 3.6. Синхронизация

- **Файл.** Скачать бэкап (`1kzariki-backup.json`) и загрузить бэкап.
- **Облако (GitHub Gist).** Поле для personal access token (scope `gist`), кнопки «Сохранить токен», «Выгрузить», «Загрузить». После первой выгрузки показывается id созданного приватного gist.

## 4. Конфигурация правил

Каждая партия хранит свой набор правил. Значения по умолчанию — из пресета «Классика».

| Настройка | Поле | Варианты | По умолчанию | Что делает |
|---|---|---|---|---|
| Цель | `target` | 1000 | 1000 | Сколько набрать для победы. |
| Бочка | `barrelAt` | 850 · 880 · без (1000) | 880 | Порог заморозки счёта. «Без бочки» = 1000 (выключено). |
| Шкала очков | `scale` | classic · fast | classic | `classic`: 1=10, 5=5. `fast`: 1=100, 5=50. Влияет на быстрые кнопки ввода и тексты правил (счёт вписывается вручную). |
| Ровно 1000 | `exact` | да · нет | да | `да` — перебор не засчитывается; `нет` — победа при ≥1000. |
| Ямы | `pitLines` | независимые тумблеры: 200–300 и 600–700 (+ «нет») | обе вкл | Фиксированные промежутки-ловушки; каждый включается отдельно, «нет» гасит все. Маркеры в таблице (информативно). |
| Штраф за болт | `samosval` | нет · −50 · −100 | нет | Вычет за каждый кон в ноль (болт). |
| Самосвал (555) | `dump555` | выкл · 1 · 2 · 3 · всегда | 3 | Счёт встал ровно на 555 — сбрасывается в ноль (автоматически). Число = сколько раз срабатывает за партию, потом перестаёт. |
| 3 болта подряд | `bolt3` | нет · −50 · −100 | −100 | Вычет за три болта кряду; счётчик болтов обнуляется. |
| Обгон | `overtake` | нет · −50 · −100 | −50 | Догнал соперника точно в его счёт — он уходит в минус на эту сумму. |
| Порог входа | `entry` | без · 100 · 120 | 120 | Сколько надо набрать за кон, чтобы открыть счёт. |
| Режим входа | `entryMode` | once · each | once | `once` — зашёл один раз и в игре до конца (даже из минуса). `each` — любой кон меньше порога обнуляет в ноль, заходишь заново. Виден только при включённом пороге. |

### Пресеты

|---|---|---|---|---|---|---|---|---|---|---|
| Классика | 880 | classic | да | 200–300, 600–700 | нет | 3 | −100 | −50 | 120 | once |
| Дворовая | 850 | classic | да | 200–300, 600–700 | нет | всегда | −100 | −100 | 120 | each |

Точная семантика подсчёта (как именно применяются штрафы, заход и обгон) — в [ARCHITECTURE.md §5](ARCHITECTURE.md#5-движок-подсчёта).

## 5. Ввод и правка счёта

- **Запись кона.** Тап по клетке открывает цифровую модалку: в шапке текущий ввод, кнопки `⌫` (стереть символ) и очистки; цифровая клавиатура; быстрые комбинации (зависят от шкалы); кнопка «Болт · 0»; «Записать». Самосвал-555 считается автоматически по сумме, отдельной кнопки нет.
- **Правка.** Тап по уже заполненной клетке открывает ту же модалку с текущим значением. Есть «Очистить ход» (вернуть клетку в пустое состояние).
- **Новый кон.** Кнопка «+ Новый кон» добавляет пустую строку. Если конов ещё нет, по тапу на «+» в пустой клетке создаётся первый кон.
- **Удаление кона.** Кнопка «Править коны» включает режим, в котором в колонке «кон» вместо номеров — красные ✕. Тап → подтверждение «Удалить кон N» → кон стирается у всех игроков, итоги пересчитываются. «Готово» выключает режим.

## 6. Имя игрока (клавиатура)

Модалка с раскладками **АБВ (ЙЦУКЕН) / ABC (QWERTY) / 123**, регистром (⇧), шапкой с текущим вводом, `⌫` и «стереть всё». По умолчанию подставляется «Игрок N».

## 7. Нефункциональные требования

- Устанавливается как PWA и работает офлайн: service worker кэширует оболочку и CDN, на навигацию при офлайне отдаётся `index.html`. Манифест даёт установку на домашний экран и запуск в standalone.
- Состояние переживает перезагрузку страницы в обычном браузере. В песочнице без доступа к `localStorage` приложение не падает, но состояние живёт только в сессии.
- Никаких внешних запросов, кроме CDN (скрипты/шрифты) и GitHub API (только при ручной синхронизации).

## 8. Ограничения текущей версии

- `scale` влияет только на быстрые кнопки и тексты правил, не на авто-подсчёт комбинаций (очки за кон вписываются вручную).
- Ямы — информативные маркеры, без авто-штрафа.
- Правило «один на бочке — второй слетает» не энфорсится автоматически (подсветка есть, списания нет).
- Подтверждающий бросок (`confirm`) зарезервирован, в движке не применяется.
- Обгон срабатывает только при точном равенстве счёта.
- Коны, сыгранные до захода или сгоревшие при обнулении, в таблице показываются обычным цветом.
